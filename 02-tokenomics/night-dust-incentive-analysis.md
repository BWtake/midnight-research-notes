# NIGHT / DUST Incentive Analysis

## Overview
This note extends the basic NIGHT / DUST model into an incentive and value capture framework.

## 1. System Structure

Midnight uses a dual-layer model:

- NIGHT → economic / governance layer
- DUST → resource / execution layer

This separation introduces a decoupled incentive system.

---

## 2. Value Capture (NIGHT)

NIGHT may capture value through:

- Governance participation
- Ecosystem coordination
- Access to DUST generation
- Builder / validator incentives

Key question:
→ Does holding NIGHT provide structural economic advantage?

---

## 3. Demand Drivers

### DUST Demand

DUST demand is derived from:

- Transaction execution
- Smart contract usage
- Privacy-preserving computation

Therefore:

DUST demand ∝ network usage

---

### NIGHT Demand

NIGHT demand may be driven by:

- Need to generate DUST
- Governance participation
- Ecosystem positioning
- Speculative demand

---

## 4. Supply Pressure

Potential supply sources:

- Glacier Drop distribution
- Early participant allocation
- Ecosystem incentives
- Secondary market liquidity

Key risk:
→ Early supply overhang vs real demand

---

## 5. Structural Hypothesis

If designed correctly:

- Users consume DUST
- Builders need DUST
- DUST requires NIGHT (directly or indirectly)

Then:

→ NIGHT becomes an upstream asset of network activity

---

## 6. Critical Unknowns

- DUST generation rate
- Conversion mechanism (NIGHT → DUST)
- Supply unlock schedule
- Fee model design
- Builder incentive structure

These variables will determine whether NIGHT captures value or not.

---

## 7. Preliminary Evaluation

Strengths:

- Separation of economic layer and execution layer
- Potential privacy-preserving fee abstraction
- Flexibility in economic design

Risks:

- Weak linkage between NIGHT and DUST
- Excessive supply before demand
- Low real usage of network

---

## 8. Research Direction

Next steps:

- Quantify DUST generation mechanics
- Track actual network usage
- Observe builder adoption
- Monitor token distribution behavior
