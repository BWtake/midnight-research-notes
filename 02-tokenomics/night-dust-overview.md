# NIGHT and DUST Overview

Research notes about the Midnight Network token model.

The Midnight ecosystem uses a dual-token structure composed of NIGHT and DUST.

## NIGHT

NIGHT is expected to function as the primary utility token of the Midnight ecosystem.

Possible roles include:

- Governance participation
- Ecosystem incentives
- Alignment of builders and community
- Economic coordination across Midnight applications

## DUST

DUST represents the network resource layer.

Possible functions include:

- Transaction resource accounting
- Smart contract execution resource
- Privacy-preserving computation costs

This separation may allow Midnight to design privacy-preserving economic incentives without directly exposing transaction-level economic data.

## Observations

The dual-component token model appears designed to separate:

- Governance and ecosystem incentives (NIGHT)
- Network resource accounting (DUST)

This structure may support scalable privacy-focused computation.

Further research will track:

- Glacier Drop distribution
- Incentive design
- Builder ecosystem growth
